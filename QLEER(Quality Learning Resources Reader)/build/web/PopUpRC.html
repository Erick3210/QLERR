¡Calculo de la Informatividad!

Uno de los sub-atributos para medir la reusabilidad de un recurso de aprendizaje es la informatividad.

Informatividad: Según el modelo de calidad implementado en este medidor, indica que la informatividad de un recurso de aprendizaje, es la sumatoria de los valores de las etiquetas schoolLevel, domain, topic, sub, contentType, SWATypes, y la etiqueta adicional que engloba todas las demas SWAInformation.

Donde ¡Informatividad! es la sumatoria de los valores de las etiquetas presentes en el WSDL del servicio web de aprendizaje (SWA) utilizando NuSOAP, estas etiquetas son 5: (types, message, portType, binding y service).

E: Es el valor de la etiqueta del WSDL de un servicio Web normal, donde se asigna 1 si existe la etiqueta de lo contrario se asigna un 0.

n + 1: Es el total de etiquetas que deben estar presentes en el WSDL de un SWA, donde n toma el valor de 5, ya que es el número de etiquetas principales en el WSDL de un servicio Web normal, el valor de 1 es para incluir el valor del conjunto de etiquetas adicionales, por lo tanto n + 1 = 6.

Para obtener el valor ∆E, que es el valor del conjunto de las etiquetas adicionales, se revisa como primera instancia que exista la etiqueta SWAInformation en el WSDL, si la etiqueta existe se aplica la métrica definida en la sig. ecuación, esto determina si todas las etiquetas adicionales están presentes [1].

∆E: Es el valor de la sumatoria de las etiquetas adicionales, estas etiquetas son 6: SchoolLevel, Domain, Topic, Sub, ContentType y SWATypes.

E_A: Es el valor de la etiqueta adicional (SWAInformation), donde se asigna 1 si existe la etiqueta de lo contrario se asigna un 0.

m: Es el número total de las etiquetas adicionales, por lo tanto m = 6.

En caso de que la etiqueta SWAInformation no exista, ∆E automáticamente tomará el valor de 0, debido a que la etiqueta SWAInformation engloba a todas las demás, tal y como se ve en la siguiente figura.
